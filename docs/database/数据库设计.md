# FitTracker 数据库设计文档

> 本地数据库采用 HarmonyOS RelationalStore（底层是 SQLite）
> 零基础说明：数据库就像一个 Excel 工作簿，每张"表"是一个 Sheet，每行是一条记录

---

## 数据库概览

| 表名 | 用途 | 对应功能 |
|------|------|---------|
| `users` | 用户基本信息 | 个人中心 |
| `checkin_records` | 每日打卡记录 | 打卡系统 |
| `workout_sessions` | 训练会话（每次健身） | 训练记录 |
| `workout_sets` | 训练组（每个动作的每一组） | 训练记录详情 |
| `weight_records` | 体重记录 | 体重图表 |
| `diary_entries` | 健身感受日记 | 感受日记 |
| `exercises` | 动作库（内置+自定义） | 动作教学 |
| `workout_templates` | 训练模板 | 训练计划向导 |
| `nutrition_logs` | 营养/饮食记录 | 饮食记录 |

## 详细表结构

### 1. 用户表 `users`

```sql
CREATE TABLE users (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,  -- 用户唯一ID（自动+1）
    name        TEXT    NOT NULL,                   -- 用户昵称
    avatar_uri  TEXT,                               -- 头像图片路径
    gender      TEXT    CHECK(gender IN ('male','female','other')),
    birth_date  TEXT,                               -- 生日，格式：2000-01-15
    height_cm   REAL,                               -- 身高（厘米）
    target_weight_kg REAL,                          -- 目标体重（千克）
    unit_system TEXT    DEFAULT 'metric'            -- 'metric'公制 / 'imperial'英制
                        CHECK(unit_system IN ('metric','imperial')),
    huawei_id   TEXT,                               -- 华为账号ID（云同步用）
    created_at  TEXT    DEFAULT (datetime('now')),  -- 创建时间
    updated_at  TEXT    DEFAULT (datetime('now'))   -- 最后修改时间
);
```

**字段解释（对零基础说明）：**
- `PRIMARY KEY AUTOINCREMENT`：像 Python 里的自动编号，第一条记录 id=1，第二条 id=2，以此类推
- `TEXT NOT NULL`：这一格必须填，不能留空（像函数里的必填参数）
- `CHECK(...)`：相当于 Python 的 `if value not in [...]: raise ValueError`
- `datetime('now')`：自动填入当前时间，不需要手动写

### 2. 打卡记录表 `checkin_records`

```sql
CREATE TABLE checkin_records (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id         INTEGER NOT NULL,              -- 关联 users.id
    checkin_date    TEXT    NOT NULL UNIQUE,       -- 打卡日期，格式：2026-05-17
                                                   -- UNIQUE 保证同一天只能打一次
    is_checked      INTEGER NOT NULL DEFAULT 0,    -- 1=已打卡，0=未打卡（SQLite 没有 bool）
    checkin_time    TEXT,                          -- 实际打卡时间，如 "20:30"
    streak_count    INTEGER DEFAULT 0,             -- 当前连续打卡天数
    note            TEXT,                          -- 当天备注
    created_at      TEXT    DEFAULT (datetime('now')),
    
    FOREIGN KEY (user_id) REFERENCES users(id)    -- 外键：确保 user_id 在 users 表中存在
);

-- 索引：加速按日期查询（类似书的目录）
CREATE INDEX idx_checkin_date ON checkin_records(checkin_date);
CREATE INDEX idx_checkin_user ON checkin_records(user_id);
```

**连续打卡（Streak）计算逻辑：**
```
今天打卡 AND 昨天也打卡 → streak = 昨天的 streak + 1
今天打卡 AND 昨天没打卡 → streak = 1（重新开始）
```

### 3. 训练会话表 `workout_sessions`

```sql
-- 每次完整的健身记录为一个 session（会话）
-- 例如：5月17日去健身房练了1小时，就是一个 session
CREATE TABLE workout_sessions (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id         INTEGER NOT NULL,
    session_date    TEXT    NOT NULL,              -- 训练日期，格式：2026-05-17
    start_time      TEXT,                          -- 开始时间，如 "19:00"
    end_time        TEXT,                          -- 结束时间，如 "20:30"
    duration_min    INTEGER,                       -- 总时长（分钟），可自动计算
    total_volume_kg REAL DEFAULT 0,                -- 总训练量（所有重量×组数×次数之和）
    mood_score      INTEGER DEFAULT 3              -- 训练前状态评分 1-5
                    CHECK(mood_score BETWEEN 1 AND 5),
    notes           TEXT,                          -- 整体训练备注
    template_id     INTEGER,                       -- 使用的训练模板ID（可为空）
    created_at      TEXT    DEFAULT (datetime('now')),
    
    FOREIGN KEY (user_id)     REFERENCES users(id),
    FOREIGN KEY (template_id) REFERENCES workout_templates(id)
);

CREATE INDEX idx_session_date ON workout_sessions(session_date);
CREATE INDEX idx_session_user ON workout_sessions(user_id);
```

### 4. 训练组表 `workout_sets`

```sql
-- 每个 session 下面有多个动作，每个动作有多组
-- 例如：卧推 4组 × 10次 × 60kg → 4行记录
CREATE TABLE workout_sets (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id      INTEGER NOT NULL,              -- 关联 workout_sessions.id
    exercise_id     INTEGER NOT NULL,              -- 关联 exercises.id（动作库）
    exercise_name   TEXT    NOT NULL,              -- 冗余存储动作名（防止动作库被删导致历史丢失）
    muscle_group    TEXT,                          -- 主要肌肉群：chest/back/legs/shoulders/arms/core
    set_number      INTEGER NOT NULL,              -- 第几组（1,2,3,4...）
    reps            INTEGER,                       -- 次数（有氧项目可为空）
    weight_kg       REAL    DEFAULT 0,             -- 重量（千克）
    duration_sec    INTEGER,                       -- 时长（秒，用于有氧/平板支撑等）
    distance_km     REAL,                          -- 距离（公里，用于跑步/骑行）
    rest_sec        INTEGER DEFAULT 60,            -- 休息时间（秒）
    is_warmup       INTEGER DEFAULT 0,             -- 1=热身组，不计入总量
    rpe             INTEGER,                       -- 主观用力程度 1-10（Rate of Perceived Exertion）
    created_at      TEXT    DEFAULT (datetime('now')),
    
    FOREIGN KEY (session_id)  REFERENCES workout_sessions(id) ON DELETE CASCADE,
    FOREIGN KEY (exercise_id) REFERENCES exercises(id)
);
-- ON DELETE CASCADE：删掉 session 时，自动删掉它下面所有的 sets（级联删除）

CREATE INDEX idx_sets_session ON workout_sets(session_id);
CREATE INDEX idx_sets_exercise ON workout_sets(exercise_id);
```

### 5. 体重记录表 `weight_records`

```sql
CREATE TABLE weight_records (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id         INTEGER NOT NULL,
    record_date     TEXT    NOT NULL,              -- 记录日期，格式：2026-05-17
    weight_kg       REAL    NOT NULL,              -- 体重（千克，统一存 kg，显示时换算）
    body_fat_pct    REAL,                          -- 体脂率（%，可选）
    muscle_mass_kg  REAL,                          -- 肌肉量（千克，可选，需体脂秤）
    bmi             REAL,                          -- BMI（自动计算：体重/身高²）
    note            TEXT,
    measured_at     TEXT,                          -- 具体测量时间，如 "07:30"（早上空腹最准）
    created_at      TEXT    DEFAULT (datetime('now')),
    
    UNIQUE(user_id, record_date),                  -- 同一天同一用户只能有一条记录
    FOREIGN KEY (user_id) REFERENCES users(id)
);

CREATE INDEX idx_weight_date ON weight_records(record_date);
```

**BMI 计算（在代码里自动算）：**
```python
# Python 写法（方便理解，实际用 ArkTS）
height_m = height_cm / 100          # 身高转换为米
bmi = weight_kg / (height_m ** 2)   # BMI 公式

# 评级
# < 18.5  → 偏瘦
# 18.5-24 → 正常
# 24-28   → 超重
# > 28    → 肥胖
```

### 6. 感受日记表 `diary_entries`

```sql
CREATE TABLE diary_entries (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id         INTEGER NOT NULL,
    entry_date      TEXT    NOT NULL,              -- 日记日期
    session_id      INTEGER,                       -- 关联的训练会话（可为空，休息日也可写）
    energy_level    INTEGER DEFAULT 3              -- 体力状态 1-5（1很累 5很亢奋）
                    CHECK(energy_level BETWEEN 1 AND 5),
    mood_emoji      TEXT,                          -- 心情标签：happy/tired/motivated/sore/proud
    content         TEXT,                          -- 正文（自由文字）
    photo_uris      TEXT,                          -- 照片路径列表（JSON数组字符串存储）
                                                   -- 例如：'["file://...jpg","file://...jpg"]'
    is_public       INTEGER DEFAULT 0,             -- 0=私密，1=可分享（预留社交功能）
    created_at      TEXT    DEFAULT (datetime('now')),
    
    UNIQUE(user_id, entry_date),
    FOREIGN KEY (user_id)    REFERENCES users(id),
    FOREIGN KEY (session_id) REFERENCES workout_sessions(id)
);
```

### 7. 动作库表 `exercises`

```sql
CREATE TABLE exercises (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    name            TEXT    NOT NULL,              -- 动作名称，如"卧推"
    name_en         TEXT,                          -- 英文名，如"Bench Press"
    muscle_group    TEXT    NOT NULL,              -- 主要肌群
    secondary_muscles TEXT,                        -- 次要肌群（JSON数组）
    equipment       TEXT,                          -- 器械：barbell/dumbbell/machine/bodyweight/cable
    exercise_type   TEXT    DEFAULT 'strength'     -- 类型：strength力量/cardio有氧/flexibility柔韧
                    CHECK(exercise_type IN ('strength','cardio','flexibility')),
    description     TEXT,                          -- 动作要领说明
    video_uri       TEXT,                          -- 教学视频路径或 URL
    gif_uri         TEXT,                          -- 动作 GIF 图路径
    is_builtin      INTEGER DEFAULT 1,             -- 1=内置动作，0=用户自定义
    is_active       INTEGER DEFAULT 1,             -- 1=启用，0=用户隐藏（软删除）
    created_at      TEXT    DEFAULT (datetime('now'))
);

-- 预置数据示例（部分常见动作）：
-- 胸部：卧推、上斜哑铃飞鸟、双杠臂屈伸、绳索夹胸
-- 背部：引体向上、坐姿划船、硬拉、高位下拉
-- 腿部：深蹲、腿举、腿弯举、提踵
-- 肩部：哑铃侧平举、推举、面拉
-- 手臂：二头弯举、三头下压、锤式弯举
-- 核心：平板支撑、卷腹、俄罗斯转体
-- 有氧：跑步、骑行、跳绳、划船机
```

### 8. 训练模板表 `workout_templates`

```sql
CREATE TABLE workout_templates (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    name            TEXT    NOT NULL,              -- 模板名称，如"胸背训练日A"
    goal            TEXT,                          -- 目标：fat_loss减脂/muscle_gain增肌/strength增力
    difficulty      TEXT,                          -- beginner初级/intermediate中级/advanced高级
    estimated_min   INTEGER,                       -- 预计时长（分钟）
    description     TEXT,
    is_builtin      INTEGER DEFAULT 0,             -- 1=系统预置，0=用户创建
    user_id         INTEGER,                       -- 用户创建的模板归属（系统模板为 NULL）
    exercises_json  TEXT,                          -- 模板包含的动作（JSON格式存储）
    created_at      TEXT    DEFAULT (datetime('now'))
);

-- exercises_json 示例：
-- [
--   {"exercise_id": 1, "sets": 4, "reps": 10, "weight_pct": 0.75, "rest_sec": 90},
--   {"exercise_id": 5, "sets": 3, "reps": 12, "weight_pct": 0.70, "rest_sec": 60}
-- ]
```

### 9. 营养记录表 `nutrition_logs`

```sql
CREATE TABLE nutrition_logs (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id         INTEGER NOT NULL,
    log_date        TEXT    NOT NULL,              -- 日期
    meal_type       TEXT    CHECK(meal_type IN ('breakfast','lunch','dinner','snack')),
    food_name       TEXT    NOT NULL,              -- 食物名称
    amount_g        REAL,                          -- 克数
    calories        REAL,                          -- 热量（千卡）
    protein_g       REAL,                          -- 蛋白质（克）
    carbs_g         REAL,                          -- 碳水化合物（克）
    fat_g           REAL,                          -- 脂肪（克）
    created_at      TEXT    DEFAULT (datetime('now')),
    
    FOREIGN KEY (user_id) REFERENCES users(id)
);

CREATE INDEX idx_nutrition_date ON nutrition_logs(log_date, user_id);
```

## 常用查询示例（用 Python 注释帮你理解 SQL）

```sql
-- 1. 查询最近30天的体重趋势
-- Python 等价：[r for r in records if r.date >= 30天前]
SELECT record_date, weight_kg, bmi
FROM weight_records
WHERE user_id = 1
  AND record_date >= date('now', '-30 days')
ORDER BY record_date ASC;

-- 2. 计算当前连续打卡天数
-- 找最近一次断签的日期，今天到那之间的天数就是 streak
SELECT COUNT(*) as streak
FROM checkin_records
WHERE user_id = 1
  AND is_checked = 1
  AND checkin_date >= (
    SELECT MAX(checkin_date)
    FROM checkin_records
    WHERE user_id = 1 AND is_checked = 0
  );

-- 3. 查询某个动作的历史最佳重量
-- Python 等价：max(set.weight for set in sets if set.exercise_id == x)
SELECT MAX(weight_kg) as personal_best, MAX(reps) as best_reps
FROM workout_sets
WHERE exercise_id = 1
  AND is_warmup = 0;

-- 4. 某月训练总天数统计
SELECT COUNT(DISTINCT session_date) as workout_days
FROM workout_sessions
WHERE user_id = 1
  AND strftime('%Y-%m', session_date) = '2026-05';
```
