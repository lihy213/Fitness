## 整体目录结构

```
FitTracker/                        ← 项目根目录（APP 叫 FitTracker）
├── AppScope/                      ← 全局配置（整个 APP 的"身份证"）
│   └── resources/base/media/      ← APP 图标、启动图
│
├── entry/                         ← 主模块（所有功能代码都在这里）
│   └── src/main/
│       ├── ets/                   ← ArkTS 代码目录（相当于 Python 的 .py 文件们）
│       │   ├── pages/             ← 【页面】用户看到的每一个界面
│       │   │   ├── HomePage.ets   ← 首页（打卡 + 今日概览）
│       │   │   ├── WorkoutPage.ets← 训练记录页
│       │   │   ├── WeightPage.ets ← 体重图表页
│       │   │   ├── DiaryPage.ets  ← 感受日记页
│       │   │   └── ProfilePage.ets← 个人中心页
│       │   │
│       │   ├── components/        ← 【组件】可复用的 UI 零件（像乐高积木）
│       │   │   ├── CheckinCard.ets← 打卡卡片组件
│       │   │   ├── WeightChart.ets← 体重折线图组件
│       │   │   ├── WorkoutItem.ets← 训练条目组件
│       │   │   └── MoodPicker.ets ← 心情选择器组件
│       │   │
│       │   ├── viewmodels/        ← 【ViewModel】页面的"大脑"，处理页面逻辑
│       │   │   ├── HomeViewModel.ets
│       │   │   ├── WorkoutViewModel.ets
│       │   │   └── WeightViewModel.ets
│       │   │
│       │   ├── services/          ← 【服务层】业务规则（打卡逻辑、云同步等）
│       │   │   ├── CheckinService.ets  ← 打卡业务：计算连续天数等
│       │   │   ├── WorkoutService.ets  ← 训练业务：模板套用、历史最佳
│       │   │   ├── WeightService.ets   ← 体重业务：BMI计算、趋势分析
│       │   │   └── SyncService.ets     ← 云同步业务
│       │   │
│       │   ├── database/          ← 【数据库层】数据的读取和存储
│       │   │   ├── dao/           ← DAO = Data Access Object，专门负责增删改查
│       │   │   │   ├── CheckinDao.ets
│       │   │   │   ├── WorkoutDao.ets
│       │   │   │   └── WeightDao.ets
│       │   │   └── entities/      ← 数据结构定义（相当于 Python 的数据类）
│       │   │       ├── WorkoutSession.ets
│       │   │       ├── WeightRecord.ets
│       │   │       └── CheckinRecord.ets
│       │   │
│       │   ├── models/            ← 【模型】纯数据结构（无业务逻辑）
│       │   │   └── ExerciseModel.ets ← 动作库数据结构
│       │   │
│       │   ├── utils/             ← 【工具函数】通用小工具（日期格式化、单位转换等）
│       │   │   ├── DateUtils.ets
│       │   │   ├── UnitUtils.ets  ← kg/lb 转换，cm/inch 转换
│       │   │   └── ChartUtils.ets ← 图表数据格式化
│       │   │
│       │   └── constants/         ← 【常量】固定不变的配置值
│       │       └── AppConstants.ets ← 颜色、字号、API 地址等
│       │
│       └── resources/             ← 【资源文件】图片、字符串、颜色
│           └── base/
│               ├── element/       ← 字符串(string.json)、颜色(color.json)
│               ├── media/         ← 图片资源
│               └── profile/       ← 路由配置(main_pages.json)
│
├── docs/                          ← 【项目文档】（这个目录就是你现在在看的）
│   ├── architecture/              ← 架构设计文档
│   ├── database/                  ← 数据库设计文档
│   └── ui-wireframes/             ← UI 线框图
│
└── .git/                          ← Git 版本控制（自动管理，不要手动改）
```

## 理解 MVVM 架构（用你懂的语言解释）

你知道 Python 函数里，有时候把"读取数据"、"计算逻辑"、"打印输出"混在一起写会很乱。
MVVM 就是把这三件事分开放：

```
用 Python 类比理解：

View（页面）     = print()         ← 只负责显示
ViewModel        = 计算函数        ← 只负责处理逻辑
Model/Database   = 文件/变量       ← 只负责存数据
```

**好处**：改界面不影响逻辑，改逻辑不影响数据，三个人可以同时写三个部分。

## 数据流向（数据怎么从数据库流到屏幕）

```
用户点击按钮
    ↓
Page（页面）调用 ViewModel 的方法
    ↓
ViewModel 调用 Service（业务规则处理）
    ↓
Service 调用 DAO（数据库操作）
    ↓
DAO 从 RelationalStore（本地数据库）读取数据
    ↓
数据原路返回，页面自动更新显示
```
